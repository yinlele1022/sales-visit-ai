# 销售拜访 AI 助手

> **不只是记录工具，是帮销售 Leader 放大决策认知的 AI 参谋。**

基于 DeepSeek/Qwen 双模型链，将销售人员口述的拜访记录自动转化为结构化 CRM 数据，支持置信度自动分流、飞书实时通知、三层安全网兜底。

---

## 一句话说清

销售在浏览器粘贴一段拜访记录 → AI 提取 14 个结构化字段（含战略洞察） → 高置信度自动入库，低置信度人工确认 → 飞书推送卡片 → CSV 写入 + 审计日志 + 记忆更新。

---

## 核心功能

| 功能 | 说明 |
|------|------|
| 多模态输入 | 文本粘贴 / 拖拽上传 .txt/.csv/.docx |
| LLM 双模型链 | DeepSeek 优先 → Qwen 兜底，零单点故障 |
| 14 字段提取 | 销售/客户/联系人/议题/疑虑/竞对/下一步/风险/战略洞察等 |
| LLM 自省 | 逐字段置信度评估，幻觉/遗漏/逻辑错误自动检测 |
| 置信度自动分流 | \>0.85 自动入库，≤0.85 Web 页面确认/修改/取消 |
| 飞书卡片通知 | 三秒摘要 + 完整字段 + 战略洞察，绿色/蓝色双模式 |
| 三层安全网 | ① LLM 自省 → ② 业务规则校验 → ③ 人工复核兜底 |
| 客户记忆 | 历史拜访记录注入 Prompt，从"单次处理"变为"连续跟踪" |
| 审计日志 | Excel 留痕，含归因标签（大脑/工具/输入/规则/正常） |
| 反馈进化接口 | `feedback_loop.py` 预留 Few-shot 回流管线 |

```

             ▼
┌─ 安全网 ───────────────────────┐

             ▼
       置信度 > 0.85？

       └─ 否 → Web 确认/修改/取消
             │
             ▼
┌─ 写入层 ───────────────────────┐

└────────────────────────────────┘
```

## 快速开始

### 1. 环境准备

```bash
pip install -r requirements.txt
```

### 2. 配置 API Key

编辑 `.env` 文件：

```env
DEEPSEEK_API_KEY=sk-xxx
QWEN_API_KEY=sk-xxx
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
FEISHU_USER_ID=xxx
```

### 3. 启动

```bash
python web_app.py
```

或双击 `start_web.bat`。

浏览器打开 `http://localhost:5000`。

## 设计理念

这个项目的三个设计原则：

1. **不为 AI 而 AI**——每个功能都有明确的业务价值，战略洞察让系统从"记录"升级为"参谋"
2. **不假装 AI 不会错**——三层安全网 + 置信度分流 + 人工兜底，承认不确定性才是工程成熟的标志
3. **不满足于"能用"**——三秒摘要、归因标签、反馈接口，每个细节都在表达"我想过运维和进化"

---

## 进化路径（预留接口说明）

- **不确定性分流** (`uncertainty_type`)：区分 aleatoric（原文没提）和 epistemic（模型没抽对），不同处理策略
- **反馈学习**：`feedback_loop.py` 将人工修正格式化为 Few-shot 示例回流 Prompt
- **系统健康日报**：自动生成通过率/置信度趋势/异常案例报告
- **多模态输入**：支持语音转写、图片 OCR、飞书消息直连
